Kafka Search Pipeline (stage by stage)



	•	POST /search/query — submit (async path)
	•	POST /search/query/sync — fast-path BM25-only
	•	GET /search/results/{request_id} — poll result
	•	GET /search/status/{request_id} — lightweight status
	•	WS /ws/search/{request_id} — real-time push
	•	POST /search/feedback — relevance signal for your eval loop
	•	GET /health, GET /ready

Kafka Processors & Topics

Topics: search.request.raw → search.request.enriched → search.candidates.fused → search.results.reranked → search.response.final, plus search.audit.log and per-stage *.dlq

Processors: Query Understanding → Retrieval (fan-out) → Fusion (RRF) → Reranking → Synthesis → Response Writer
